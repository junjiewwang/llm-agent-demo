# ThinkingPanel 思考过程面板优化方案

## 一、现状分析

### 当前布局结构

```
┌─────────────────────────────────────────────────┐
│ [脉动点] 📋 计划执行 (2/4 步完成, 8 次工具调用) [▼] │  ← 标题栏
├─────────────────────────────────────────────────┤
│ 📋 已生成执行计划 (4 步)                          │  ← plan_created
│ ▶️ 步骤 1/4: 检查集群节点状态                [▼]  │  ← step_start（可折叠）
│   │ 🔄 第 1/10 轮思考                            │     ← children（缩进+左竖线）
│   │ 🔧 调用工具: kubectl | {"action":"get"...}   │
│   │ ✅ 结果 (1200ms): NAME   STATUS   ROLES...  │
│ ✅ 步骤 2/4: 查看 Pod 运行状态               [▼]  │
│   │ ...                                          │
│ ⏳ 思考中...                                      │
└─────────────────────────────────────────────────┘
```

### 存在的问题

| # | 问题 | 影响 |
|---|------|------|
| 1 | **纯文本流水账** | 所有事件只是一行文字，`🔧 调用工具: kubectl \| {"action":"get"...}`，信息密度低、可读性差 |
| 2 | **工具调用缺乏结构化** | 工具名、参数、结果混在一行纯文本中，JSON 参数被截断到 80 字符后难以理解 |
| 3 | **视觉层次扁平** | 除 Plan 模式步骤有分组外，ReAct 模式下所有事件视觉权重相同，没有主次 |
| 4 | **缺乏状态动画** | 执行中的事件没有进度感，已完成的事件没有完成反馈 |
| 5 | **工具类型无差异化** | kubectl、file_reader、web_search 等不同工具没有各自的图标和配色，无法快速区分 |
| 6 | **结果不可展开** | 工具返回结果只展示 80 字符预览，无法查看完整输出 |

## 二、优化方案设计

### 2.1 工具调用卡片化（核心改进）

将 `tool_call` + 对应的 `tool_result` 配对渲染为**结构化卡片**，替代当前的两行纯文本。

**卡片布局：**

```
┌─────────────────────────────────────────────────┐
│ 🖥️ kubectl                             ✅ 1.2s │  ← 工具图标 + 名称 + 状态 + 耗时
│ ┌─────────────────────────────────────────────┐ │
│ │ action: get                                 │ │  ← 参数区（关键字段提取展示）
│ │ resource: pods                              │ │
│ │ namespace: default                          │ │
│ └─────────────────────────────────────────────┘ │
│ ▸ 查看结果                                       │  ← 结果折叠区（默认收起，点击展开）
└─────────────────────────────────────────────────┘
```

**状态标记：**

| 状态 | 视觉 |
|------|------|
| 执行中 | 右上角旋转 spinner + "执行中..." |
| 成功 | ✅ + 耗时（绿色） |
| 失败 | ❌ + 耗时（红色） |

**参数展示逻辑：**

- 将 `tool_args` 的 JSON 按 key-value 逐行展示（非 JSON.stringify 截断）
- 值超过 100 字符时截断并附 `...`
- 参数区使用 `bg-gray-50 dark:bg-gray-800` 浅色代码块风格

**结果折叠区：**

- 默认收起，只显示 `▸ 查看结果` 按钮
- 展开后显示完整的 `tool_result_preview`，使用 `<pre>` 等宽字体
- 结果区最大高度 `max-h-48`，超出可滚动

**配对逻辑：**

- 遍历 nodes 时，遇到 `tool_call` 后向后查找同一 `tool_name` 的 `tool_result`
- 如果找到，合并渲染为一个卡片
- 如果未找到（正在执行中），渲染为无结果状态（spinner）

### 2.2 工具图标体系

为不同工具分配专属 SVG 图标和配色，方便用户快速识别。

| 工具名 | 图标 | 配色 | 说明 |
|--------|------|------|------|
| `kubectl` | 终端/命令行图标 | `text-blue-500` | Kubernetes 管理 |
| `docker` | 容器图标 | `text-sky-500` | Docker 管理 |
| `curl` | 网络/地球图标 | `text-purple-500` | HTTP 请求 |
| `file_reader` | 文件/文档图标 | `text-emerald-500` | 文件读取 |
| `file_writer` | 铅笔/编辑图标 | `text-orange-500` | 文件写入 |
| `web_search` | 搜索/放大镜图标 | `text-cyan-500` | 网页搜索 |
| `calculator` | 计算器图标 | `text-gray-500` | 数学计算 |
| `knowledge_search` | 书本图标 | `text-indigo-500` | 知识库检索 |
| `get_current_time` | 时钟图标 | `text-gray-400` | 获取时间 |
| 其他（默认） | 齿轮图标 | `text-gray-400` | 未知工具 |

实现方式：定义 `TOOL_META` 映射表，包含 `icon`（SVG JSX）、`color`（Tailwind 色类名）、`label`（展示名）。

### 2.3 迭代轮次分隔线

将 `🔄 第 1/10 轮思考` 改为带轮次编号的**分隔线**设计，减少视觉噪音。

```
当前:
  🔄 第 1/10 轮思考
  🔧 调用工具: kubectl | {"action":"get"...}
  ✅ 结果 (1200ms): NAME   STATUS   ROLES...

优化后:
  ──── 第 1 轮 ────────────────────────────
  [kubectl 工具卡片]

  ──── 第 2 轮 ────────────────────────────
  [web_search 工具卡片]
```

实现：水平线 + 居中轮次文字 + `text-[10px] text-gray-400`。

### 2.4 Plan 模式步骤进度条

在 `plan_created` 事件后、步骤列表前，添加一个横向步骤进度条。

```
┌─────────────────────────────────────────────┐
│ 🎯 检查 Kubernetes 集群健康状态              │  ← plan.goal
│                                             │
│  ● ─── ● ─── ◐ ─── ○                       │  ← 步骤圆点连线
│  检查   查看   分析   输出                    │  ← 步骤简称（取前4字）
│  节点   Pod   日志   报告                    │
└─────────────────────────────────────────────┘
```

**圆点状态映射：**

| 状态 | 视觉 |
|------|------|
| `completed` | `●` 实心绿色 |
| `running` | `◐` 蓝色脉动 |
| `failed` | `●` 实心红色 |
| `pending` | `○` 空心灰色 |
| `skipped` | `○` 空心带删除线 |

### 2.5 摘要标题栏增强

在折叠状态下提供更丰富的概览信息。

```
当前:
  📋 计划执行 (2/4 步完成, 8 次工具调用)

优化后:
  📋 计划执行 ─── 2/4 步 · 8 次工具调用 · 3.2s
  [████████████░░░░░░░░] 50%
```

改进点：

- 添加 mini 进度条（2px 高，嵌入标题栏底部）
- 总耗时统计（从第一个事件到最后一个事件的时间差）
- 分隔符从括号改为 `·` 中点，更紧凑

## 三、改动范围

| 文件 | 改动内容 | 复杂度 |
|------|---------|--------|
| `ThinkingPanel.tsx` | 核心重构：新增 ToolCard、IterationDivider、PlanProgressBar、摘要增强 | 高 |

**不需要改动的文件：**

- 后端（`events.py`、`react_agent.py` 等）— 事件数据格式不变
- Store（`chatStore.ts`）— 数据组织逻辑不变
- 类型（`events.ts`、`chat.ts`）— 类型定义不变

**纯前端渲染层优化**，不影响数据流和后端逻辑。

## 五、实施计划

单文件改动（`ThinkingPanel.tsx`），一次性实施：

1. 新增 `TOOL_META` 工具元数据映射表（图标 + 配色）
2. 新增 `ToolCard` 组件（工具调用卡片，含参数区 + 结果折叠区）
3. 新增 `IterationDivider` 组件（轮次分隔线）
4. 新增 `PlanProgressBar` 组件（步骤进度圆点条）
5. 重构 `StepChildren` 和 ReAct 模式的事件渲染逻辑
6. 增强摘要标题栏（mini 进度条 + 紧凑格式）

改动后 `make run` 构建 + Playwright 截图验证。
